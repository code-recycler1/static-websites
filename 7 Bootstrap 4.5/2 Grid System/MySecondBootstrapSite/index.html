Grid system with flexbox

Rules for the grid system

  • Three important parts in the grid: container, row, and col
  • Containers are indicated with the class class="container" or class="container-fluid"
  • The grid system only works correctly if you use rows within a container. A row alone on the page has no effect.
  • Content is placed in columns. Columns must be in a row. Only columns can be children (child elements) in a row.
  • There are column classes for each of the desired widths, indicated with a number from 1-12. If you want four equal columns, use the class .col-sm-3 or similar (because: 4x3 = 12).
    • Difference with the flexbox system (where you would only use class="col" or class="col-sm") is that you explicitly indicate when you want columns to be stacked. Namely, when the screen becomes narrower than the breakpoint associated with the -sm-* indication (which is between 576px and 768px). Otherwise, you leave the choice to Bootstrap and the browser.
  • Column widths are calculated in percentages. This way, they always fill 100% of the width of the parent element.
  • Columns have horizontal padding to create white space between the columns
  • There are five levels for the grid. These are extra small (xs), small (sm), medium (md), large (lg), and extra large (xl). Each level has a corresponding responsive breakpoint. See: getbootstrap.com/docs/4.5/layout/grid
  • The grid levels are based on the minimum width. This means they are applied at the indicated level and all higher levels. For example:
    • the column width col-sm-4 means that this column is four columns wide on small, medium, large, and extra large screens. On extra small screens, these columns are stacked.
    • the column width col-md-6 means that this column is six columns wide on medium, large, and extra large screens. On small and extra small screens, these columns are stacked.
